# RxnNet
An AI framework for automated reaction mechanism discovery.

This is the repository of the Rxnnet program package developed by the Major group.
The code uses two external packages, which must be installed locally on your machine

1) geodesic_interpolate (https://github.com/virtualzx-nad/geodesic-interpolate)
2) ORCA quantum chemistry package (https://orca-manual.mpi-muelheim.mpg.de/contents/quickstartguide/installation.html#installation)

Before cloning this repository, you need to install Git LFS (https://git-lfs.com/)

Clone the repository:
git clone https://github.com/majordt/RxnNet.git

Add the following line to your .bashrc file:
```text
export MKL_THREADING_LAYER=GNU
```

# Generating the reaction graph

Ensure that the required keywords are provided in the parameters.py file
```text
MOLECULE_NAME			: Name of the molecule 
MOLECULE_SMILE			: SMILES string of the reactant
REACTION_LIST_NAME 		: Reaction list in the reactions.py file
MAX_PATH_LENGTH_TO_RUN	: Number of reaction steps (or depth of the reaction graph)
```
Generate the reaction tree with
```text
./rxnnet --mode=run
```
The reaction graph generated by RxnNet is saved as pickle files in Results/MOLECULE_NAME

# RxnNet Analyzer
Extract the reaction mechanism from the graph and generate geodesic paths for each step of the mechanism

Required Keywords:
```text
SOURCE_SMILE 				: SMILES of initial reactant
SMILE_TO_ANALYZE 			: SMILES of final product
MAX_PATH_LENGTH_TO_ANALYZE	: Number of reaction steps between reactant and product
TS_NUM_FRAMES				: Number of intermediate geometries to generate in the geodesic path
ALIGN_TO_PRODUCT			: True for template-based approach 
							  False for distance-restraint-based approach
NUM_CONFS_TO_GENERATE	  	: Number of conformers of each intermediate to generate
```

Run the RxnNet analyzer with
```text
./rxnnet --mode=analyze
```

TIP: If RxnNet fails to generate a path under the distance-restraint-based approach (ALIGN_TO_PRODUCT = False), try with template-based approach (ALIGN_TO_PRODUCT = True).

To run the analysis on existing data, use the values provided in parameters.py for these keywords.

RxnNet analyzer extracts the distinct paths connecting the souce and target SMILES from the graph. Each of these paths will have number of reaction steps = MAX_PATH_LENGTH_TO_ANALYZE. It then iterates through each path and generates the geodesic paths after considering stereochemistry, regiochemistry, conformaЖҹons, and atom labeling. Each RxnNet analyzer run creates a directory with the timestamp inside 'Results/MOLECULE_NAME'. A directory tree for Trichodine is shown below:

Directories set0, set1, etc., contains the geodesic paths for the distinct paths generated by RxnNet. The geodesic paths generated for five-step reaction are shown below. Here, the distance restraint-based approach in RxnNet is used (ALIGN_TO_PRODUCT = False).

```text
step_1.xyz, step_2.xyz, etc	: the geodesic paths for the chemical steps in the set.
conf_1p_to_2r.xyz			: product carbocation (CC) intermedite of step 1 and reactant CC intermediate of step 2 are the same system, but are conformationally different. This geodesic path correspond to the corresponding conformational change from step 1 product to step 2 reactant.
conf_initial_reactant.xyz 	: conformational change of the initial reactant from its global minimum conformer to the reactant in step 1.
conf_final_product.xyz		: conformation change associated with final product from product conformation of final step (step 5 in this case) to its global minimum conformer.
```

# QM module: quantum chemistry calculations

Perform nudeged elastic band (NEB) calculations with ORCA using the geodesic paths as the initial guess. 

Required Keywords:
```text
FOLDER_NAME			: directory inside Results/MOLECULE_NAME where the geodesic paths are generated. In the above example: '2026-03-18_22-04-02'
QM_SET_FOLDERS		: set (distinct path) for which QM calculations need to be performed. Can be a string ('ste0'), a list ['set0', 'set1'], or left empty. If empty, QM calculations will be performed for all sets in 'FOLDER_NAME'
CHARGE				: charge of the system.
MULTIPLICITY		: multiplicity of the system.
ORCA_PATH			: path to orca executable. '/path/to/executable/orca'
OMPI_BIN_PATH		: OpenMpi binary path (required for multiprocessing in ORCA, check ORCA documentation for compatible OpenMpi version).
OMPI_LIB_PATH		: OpenMpi library path
```
Keywords for semi-empirical calculation:
```text
SEMI_EMPIRICAL			: True/False - Do semiempirical calculations at GFNn-xTB level using ORCA-xTB interface.
SEMI_EMPIRICAL_METHOD	: xTB method. e.g. 'GFN2-xTB'
SE_INITIAL_GUESS 		: Tru/False - If True, use geodesic paths generated by RxnNet analyzer as initial guess for xTB NEB calculations.
SEMI_EMPIRICAL_KEYWORDS	: xTB keywords. Could be an empty list - []
SEMI_EMPIRICAL_NPROCS	: number processors to use for xTB calculations.
```
If the SE_INITIAL_GUESS = True, the xTB calculations are performed in the following directories:

The reactant and product optimizations are first carried out, followed by the NEB calculation. In step_1, set0_step_1_se_firstopt.inp is the ORCA input for reactant intermediate optimization. xTB level calculations are then followed by DFT NEB calculation.

Since DFT NEB calculations are expensive, it is also possible to generate only the DFT-level ORCA inputs with the flag DFT_INPUT_ONLY.

Keywords for DFT calculations:
```text
DFT					: True/False - Do DFT-level NEB with.
DFT_INITIAL_GUESS	: Tru/False - If True, use NEB minimum energy paths generated with GFN2-xTB in the previous step as initial guess for DFT NEB.
DFT_INPUT_ONLY		: True/False - If True, RxnNet only writes DFT ORCA inputs.
DFT_METHOD			: DFT functional - 'M062X'
DFT_KEYWORDS		: DFT keywords ['defgrid3']
DFT_NPROCS			: number processors to use for DFT calculations.
```
